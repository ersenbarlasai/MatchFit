# ROLE: Senior Product Architect + Antigravity Agent Designer
# PROJECT: MatchFit – Verified Coach System

You are responsible for designing and implementing the FULL coach (trainer) system inside MatchFit.

This is NOT a basic user system.
This is a VERIFIED PROFESSIONAL SYSTEM with trust, validation, and monetization.

You must build this using:
- Database-first approach (PostgreSQL / Supabase)
- .agents architecture
- Persistent memory
- Scalable verification logic

---

# 1. CORE OBJECTIVE

Design a system where:

- Users can APPLY to become a coach
- Coaches must be VERIFIED before activation
- Verified coaches can create PAID or FREE sessions
- The system PREVENTS fake coaches
- Trust, reputation, and performance are continuously evaluated

---

# 2. USER ROLES

Create role system:

- USER (default)
- COACH_PENDING
- COACH_VERIFIED
- COACH_ELITE

Role transitions must be controlled by system + admin + agents.

---

# 3. DATABASE DESIGN (MANDATORY FIRST STEP)



# 4. COACH ONBOARDING FLOW

Design multi-step onboarding:

STEP 1 → Basic Info  
STEP 2 → Identity Verification  
STEP 3 → Document Upload  
STEP 4 → Work Location  
STEP 5 → Intro Video  
STEP 6 → Application Submit  

Each step must be stored and resumable.

---

# 5. AGENT SYSTEM (.agents)

Create and configure following agents:

---

## @CoachVerificationAgent

Responsibilities:
- Validate identity documents
- Check duplicate documents
- Detect suspicious patterns
- Approve/reject documents

Rules:
- Same document cannot be reused across accounts
- Missing required documents → auto reject

---

## @GuardianAgent (Extended)

Responsibilities:
- Detect fake coaches
- Validate location authenticity
- Monitor abnormal behavior (spam sessions, unrealistic schedules)

---

## @RefereeAgent (Coach Mode)

Responsibilities:
- Validate session attendance (GPS check)
- Detect no-show (coach or user)
- Update reliability_score

---

## @CoachReputationAgent

Responsibilities:
- Calculate coach score (0–100)
- Inputs:
    - attendance rate
    - user ratings
    - cancellations
    - session completion
- Update ranking priority

---

## @MonetizationAgent

Responsibilities:
- Handle session pricing logic
- Commission calculation
- Premium visibility (boosted coaches)

---

# 6. VERIFICATION SYSTEM

Implement 3-tier verification:

- BASIC → identity verified
- CERTIFIED → documents approved
- ELITE → high rating + high reliability

UI flag must reflect verification level.

---

# 7. TRUST & SCORE SYSTEM

Each coach must have:

- reliability_score (0–100)
- rating_avg (1–5)
- session_success_rate

Score must update AFTER each session.

---

# 8. ANTI-FAKE SYSTEM

Mandatory checks:

- GPS validation for sessions
- Minimum 2 participants confirmation
- Optional photo proof
- Document duplication detection

If fraud detected:

- downgrade coach
- suspend ability to create sessions
- log event

---

# 9. MONETIZATION READY STRUCTURE

Prepare system for:

- session commission (% based)
- coach boosting (visibility ranking)
- featured coach placement

DO NOT implement payment gateway yet,
but structure must support it.

# 10. UI HOOKS (IMPORTANT)

Expose API-ready structures for:

- Coach Profile Card
- Coach Badge (Verified / Elite)
- Session Cards
- Rating Display
- Trust Score Visualization

# 11. MEMORY & LOGGING

All agent decisions must be logged in:

coach_verification_logs
